JVM监控命令

发表于 2019-03-17 | 分类于 Java基础

Sun JDK包中提供了一系列工具用于查询JVM虚拟机的状态,这些工具都非常实用.

jps

显示指定系统内所有的HotSpot虚拟机进程.

命令格式: jps [optinos]

optinos :
-q : 只输出进程Id,省略主类名称
-m : 输出虚拟机进程启动时传递给主类main()函数的参数
-l : 输出主类的全名
-v : 输出虚拟机进程启动时JVM参数

jstat

用于监视虚拟机各种运行状态信息.

命令格式: jstat [optons vmid [interval][s|ms] [count]]

vmid : 虚拟机进程Id
interval : 查询间隔时间
count : 查询次数

eg:

jstat -gc 2764 250 20 代表每250毫秒查询一次进程2764垃圾收集状况,一共查询20次.

optinos :
-class : 显示类加载的相关信息
-compiler : 显示JIT编译的相关信息
-gc : 显示GC相关的信息
-gccapacity : 显示各个代的容量以及使用情况

-gccause : 显示垃圾回收的相关信息,同时显示最后一次或当前正在发生的垃圾回收的诱因

-gcmetacapacity : 显示metaspace的大小

-gcnew : 显示新生代信息

-gcnewcapacity : 显示新生代大小和使用情况

-gcold : 显示老年代和永久代的信息

-gcoldcapacity : 显示老年代的大小

-gcutil : 显示垃圾收集信息

-printcompilation : 输出JIT编译的方法信息

jmap

用于监视虚拟机中内存情况.

命令格式: jmap [options] vmid

optinos :

-heap : 输出虚拟机堆内存相关信息

-histo[:live] : 输出虚拟机堆中类的对象占用信息,如果加了”:live”,则只输出存活的类的对象的占用信息

-clstats : 打印类加载器的统计信息

-finalizerinfo : 打印正等候回收的对象的信息

-dump: : dump虚拟机堆到文件,包含live、format=b、file,eg:jmap -dump:live,format=b,file=dump.hprof vmid

jinfo

实时查看和调整虚拟机各项参数.

命令格式: jinfo [options] vmid

jstack

用于生成虚拟机当前时刻的线程快照.

命令格式: jstack [options] vmid

optinos :

-F : 当正常输出的请求不被响应时,强制输出线程堆栈

-l : 除堆栈外,显示关于锁的附加信息

-m : 如果调用到本地方法的话,显示C/C++的堆栈
